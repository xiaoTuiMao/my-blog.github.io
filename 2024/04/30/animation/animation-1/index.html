什么是动画

浏览器中的动画是指通过 JavaScript、CSS 和 WebGL 等技术实现的网页元素的动态变化和交互效果。其中，JavaScript 可以控制 DOM 元素的属性和样式，实现例如平滑滚动、淡入淡出等动画效果；CSS 则可以通过 transition 和 animation 属性定义过渡和动画效果，如渐变、旋转、缩放等；而 WebGL 是一种基于 OpenGL ES 的 JavaScript API，可以利用 GPU 加速渲染，实现更复杂的三维和二维图形渲染，例如实时渲染的游戏场景、粒子效果等。这些技术共同构成了浏览器中丰富多彩的动画功能，为用户带来更生动、流畅的网页体验。

浏览器架构

一个 Chrome 浏览器一般会有一个 Browser 进程，一个 GPU 进程，和多个 Renderer 进程，通常每个 Renderer 进程对应一个页面。

（图片来自 Inside look at modern web browser

进程 内容
Browser 进程 控制应用程序的“chrome”部分，包括地址栏、书签、后退和前进按钮。 还处理 Web 浏览器的不可见特权部分，如网络请求和文件访问。
Renderer 渲染进程 控制网站显示标签内的任何内容。
GPU 进程 与其他进程隔离处理 GPU 任务。之所以将其分开为不同的进程，是因为 GPU 处理来自多个应用程序的请求，并在同一表面上绘制它们。
我们常常所说的GPU 加速，本质上就是利用了 GPU 独立进程，可以单独处理的能力，降低渲染进程的处理压力，从而提升性能。

页面渲染过程

这一章从浏览器的渲染过程，带出图层的概念，为后续将优化埋伏笔。

在聊动画渲染过程前，我们先聊一聊浏览器的渲染过程。我们将浏览器的渲染过程分为以下几步

  1. 解析 HTML，碰到 link，异步加载 CSS，并继续解析 HTML。此时是异步加载，不阻塞 DOMOM 生产
  2. CSS 加载完成，并生产 cssom，js如果在css 标签后面，那么 js 会等待这个 cssom 生产完成后，才会执行
  3. CSSOM 与 DOMOM 解析完成会生成一个渲染树 render-tree
  4. 将渲染树交给 layout，进行计算布局，layout-tree 与 render-tree 会有所不一样，它只会计算页面可见内容的信息包含伪类相关
  5. 布局树生产好之后，又会交给 paint 阶段，paint会进行绘制，将生产不同的图层，每个图层单独进行光栅化绘制，然后从底层开始逐级网上渲染。可通过 dev-tool 的 render 功能查看
  6. 将图层树交给合成线程，合成线程会将图层信息合并，绘制成一帧的图片。
  7. 最后再每一帧的 async 信号来临前，将这一帧的图片交给 display，最终呈现到屏幕上。

动画的渲染过程

动画是如何来的？

在聊动画的时候我们经常会听到“帧”的概念，而动画可以看做是一个连续的帧序列的组合。我们简化一下浏览器在每一帧的页面呈现：

当有动画执行时，浏览器会在屏幕的垂直信号来临之前，经过一系列的操作得到这一帧的内容，交给GPU进程完成屏幕的呈现。这些连续的帧在一段时间内的组合，就形成了我们看到的动画。

从动画渲染的角度分类

网页动画大致可以分为非合成器动画和合成器动画两类。这两类动画的特点和实现方式有所不同：

合成器动画

合成器动画是一种更现代的动画技术，主要操作的是浏览器的合成层（compositing layer）。这类动画通过改变元素的变换（transform）和透明度（opacity）等属性来实现，这些操作不会影响DOM的布局，因此性能通常更好。

常见实现：

  • CSS transform和opacity动画：直接通过CSS控制，可以利用硬件加速。
  • WebGL和Canvas：用于更高级的图形和动画效果，可以直接操作图像和绘图，但编程复杂度更高。
  • Web动画API（Web Animations API）：这是一种现代的方法，允许更细粒度和更强大的控制动画。

在选择使用非合成器动画还是合成器动画时，需要根据具体的应用场景、性能要求和开发成本来决定。合成器动画虽然在性能上有优势，但在实现复杂度上通常会更高。

非合成器动画

非合成器动画是更传统的动画形式，主要通过直接操作DOM（文档对象模型）来实现，包括修改元素的布局属性如宽度、高度和边距等。

常见实现：

  • CSS动画和过渡：使用transition和animation属性来实现简单的动画效果。
  • JavaScript动画库：如jQuery的animate()方法，可以实现更复杂的动画效果。
