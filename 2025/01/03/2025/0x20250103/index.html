React 潜在的内存泄漏风险

我们都知道在浏览器的垃圾回收机制里，会采用标记清楚的算法，浏览器扫描不被引用的对象，进行销毁。当我们使用 useCallback 或者 useMemo 的时候，在依赖没有变更的时候，会使用上一次的值，而上一次的值存在在上次 render 函数创建的闭包，因为存在引用关系，所以对应的作用域下的变量都不会得到销毁，当依赖发生变更，创建新的对象，才会对上一次的作用域里的变量进行回收。在常规场景下，仅仅只保留上次的闭包作用域还不会造成内存泄漏。但是如果我们操作不当，可能会导致本该被销毁的作用域没有销毁，从而造成内存泄漏。该系列分为两部分，第一部分讲解了使用 useCallback 这类缓存hook时，可能存在的内存泄漏风险，第二部分解析在利用 React Compiler 的情况下，可能存在的内存泄漏风险。

从 web 到 Native 使用 React

这篇文章详细介绍了如何使用 React 从 Web 开发过渡到原生应用开发，重点介绍了 React Native 的核心概念、常用组件、样式处理以及常见的开发陷阱。
React Native 核心组件：

  • View：类似于 Web 中的 div，用于布局和样式。
  • ScrollView：用于创建可滚动的视图，适用于需要滚动内容的场景。
  • Text：用于显示文本内容，所有文本都必须包裹在 Text 组件中。
  • Image：用于显示图片，支持从 URL 或本地文件加载。
  • FlatList：用于高性能地渲染长列表数据，类似于 Web 中的 array.map()。
  • TextInput：用于接受用户输入，类似于 Web 中的 。
  • TouchableOpacity 和 Pressable：用于创建可点击区域，类似于 Web 中的按钮。
  • Switch：用于创建开关组件，类似于 Web 中的 。
    样式处理：
    React Native 使用内联样式，通过 style 属性传递给组件。 虽然大多数 CSS 属性都被支持，但存在一些差异：
    没有全局样式，所有样式都是局部的。
    使用 Flexbox 进行布局，但默认值与 Web 不同，例如 flexDirection 默认为 column。
    此外，社区中有一些流行的样式库可供选择，如 NativeWind、Styled Components 和 Tamagui。
    常见陷阱：
  • 避免使用 array.map 渲染长列表：在原生应用中，直接使用 array.map 渲染长列表可能导致性能问题，建议使用 FlatList 或其他虚拟列表组件。
  • 避免使用内置的 Button 组件：React Native 的 Button 组件定制性差，建议使用 TouchableOpacity 或 Pressable 来创建按钮。
  • 避免在渲染中使用逻辑与 (&&) 运算符：直接使用 && 可能导致渲染意外的值，建议使用三元运算符进行条件渲染。
  • 不要在应用代码中存放敏感信息：与 Web 类似，避免在客户端代码中包含敏感的 API 密钥或机密信息。
    原生开发特有的概念：
  • 构建签名：在发布应用时，需要对应用进行签名，以确保其来源可信。
  • 深度链接：允许通过特定的 URL 打开应用中的特定页面。
  • 手势和触摸：移动应用中常用的交互方式，如滑动、长按等。
  • 触觉反馈：通过设备的振动反馈增强用户体验。
  • 动画：使用 React Native 的动画 API 创建流畅的用户界面。
  • 像素密度：不同设备的像素密度不同，需要考虑在不同设备上显示的清晰度。
    这篇文章为从 Web 开发转向原生应用开发的开发者提供了全面的指导，帮助他们理解 React Native 的核心概念和最佳实践。

每位开发者都应掌握的10个重要终端命令

  1. grep：用于在文件中搜索特定模式，类似于功能强大的搜索工具。
  • 基本用法：grep “pattern” filename
  • 忽略大小写：grep -i “pattern” filename
  • 统计匹配行数：grep -c “pattern” filename
  • 递归搜索：grep -r “pattern” directory/
  1. man：显示其他命令的手册页，提供详细的使用信息。
  • 用法：man command
  1. cat：用于显示文件内容、合并文件或创建新文件。
  • 显示文件内容：cat filename
  • 合并文件：cat file1 file2 > combined_file
  • 创建新文件：cat > newfile，输入内容后按 Ctrl+D 保存。
  1. head：用于查看文件的前几行，默认显示前10行。
  • 用法：head filename
  • 指定行数：head -n 5 filename
  1. awk：强大的文本处理工具，可用于模式扫描和处理，特别适合处理和分析文本文件和数据流。
  2. sed：用于在文件中执行基本的文本转换，如查找和替换。
  • 用法：sed ‘s/old/new/g’ filename
  1. tail：用于查看文件的最后几行，常用于监控日志文件。
  • 用法：tail filename
  • 实时监控：tail -f filename
  1. chmod：用于更改文件或目录的权限。
  • 用法：chmod 755 filename
  1. ssh：用于通过安全外壳协议连接到远程服务器。
  • 用法：ssh user@hostname
  1. tar：用于创建和解压归档文件。
  • 创建归档：tar -cvf archive.tar file1 file2
  • 解压归档：tar -xvf archive.tar
